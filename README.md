# FLIR camera debug tool

Программный комплекс для отладки, тестирования и управления промышленными камерами FLIR (серия Blackfly S). Инструмент предоставляет графический интерфейс для тонкой настройки сенсора, записи медиаданных и аппаратного мониторинга пропускной способности интерфейса GigE.

Архитектура построена на паттерне Model-View-Controller: высокопроизводительный бэкенд на Python (Spinnaker SDK / OpenCV) и аппаратный рендеринг интерфейса на QML.

---

## Быстрый старт

### Требования
Для развертывания приложения требуются следующие компоненты:
* Python 3.10+
* Драйверы камеры: **Spinnaker SDK** и модуль **PySpin** от Teledyne FLIR.
* Библиотеки: `PySide6` (GUI), `opencv-python` (кодеки и конвертация), `numpy` (математика цвета).

### Установка и запуск
1. Подключите камеру FLIR к гигабитному порту (GigE) или USB 3.0.
2. Клонируйте репозиторий проекта.
3. Установите зависимости:
   ```bash
   pip install PySide6 opencv-python numpy
   ```
4. Запустите точку входа:
   ```bash
   python main.py
   ```

---

## Базовая документация

Интерфейс разделен на две логические зоны:
1. **Зона видеопотока (Вьювер):** Прямая трансляция данных с матрицы без задержек (Zero-Copy).
2. **Панель управления (Настройки):** Блоки параметров сенсора, управления пресетами, записи медиа и телеметрии.

## Примеры использования (Сценарии)

### Сценарий 1: Использование для захвата фото и видео
Инструмент позволяет вести прямую запись сырых потоков данных на накопитель, минуя буфер оперативной памяти.

* **Фотофиксация:** Нажмите синюю кнопку `СНИМОК`. В появившемся системном диалоге выберите формат (`.jpg` или `.png`) и укажите путь.
* **Запись видео:** Нажмите фиолетовую кнопку `ВИДЕО`. Выберите контейнер (MP4 или AVI). Запись начнется немедленно и будет использовать аппаратный кодек (`mp4v` или `XVID`) с частотой кадров, соответствующей текущей пропускной способности сети. Для остановки нажмите красную кнопку `СТОП ЗАПИСЬ`.

### Сценарий 2: Настройка параметров изображения
Приложение предоставляет прямой доступ к аппаратным регистрам сенсора. 
* Выбор формата пикселей (Pixel Format):** Управляет режимом работы АЦП и процессора камеры:
    * **Mono8 (Ч/Б Быстрый):** Камера работает в монохромном режиме. Обеспечивает минимальную нагрузку на сеть, идеален для задач машинного зрения (поиск контуров, дефектов).
    * **RGB8 (Цвет Обработанный):** Камера аппаратно собирает цвет из RAW. Дает готовую красивую картинку, но алгоритмы самой камеры могут создавать задержку и снижать максимальный FPS.
    * **BayerRG8 (RAW Цвет):** Трансляция "сырых" данных с матрицы. Обеспечивает максимальную пропускную способность сети и частоту кадров. Расшифровка цвета выполняется программным ядром на лету.
* **Устранение артефактов движения (Motion Blur):** Снизьте `Выдержку` (Exposure) до 5000–10000 µs. Изображение станет темнее, но объекты в движении обретут резкость. Компенсируйте потерю света увеличением `Усиления` (Gain).
* **Калибровка цвета (Дебайеризация):** Для получения "честного" цвета с сенсора установите "Формат пикселей" на `BayerRG8`.
* **Баланс белого (AWB):** Активируйте тумблер `АВТО`. Запустится гибридный алгоритм "Серый мир", который будет математически выравнивать красный и синий каналы каждые 1.5 секунды. Для фиксации цвета — выключите тумблер.
* **Контрастность:** Используйте слайдер `Гамма`, чтобы вытянуть детали из теней (значения > 1.0) или сделать картинку более плотной (значения < 1.0).

### Сценарий 3: Тестирование производительности (FPS)
Для инженеров встраиваемых систем критически важен контроль пропускной способности канала. Блок телеметрии в реальном времени отображает:
* **CUR / AVG FPS:** Текущая и средняя частота кадров.
* **TARGET:** Максимально возможный FPS, который разрешает выдать камера при текущей пропускной способности сети (Traffic Shaping).

### Сценарий 4: Проверка стабильности подключения
Для диагностики кабелей и сетевых плат (GigE) используйте параметр **EFFICIENCY** в блоке телеметрии.
* **Показатель > 95% (Зеленый):** Канал стабилен, пакеты не теряются.
* **Показатель 50-90% (Оранжевый):** Происходит частичная потеря Jumbo-фреймов. Рекомендуется проверить настройки сетевого адаптера или заменить кабель.
* **Показатель < 50% (Красный):** Критический сбой линии передачи данных.
